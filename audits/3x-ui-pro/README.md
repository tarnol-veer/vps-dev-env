- файл: `x-ui-latest.sh`
- ветка: `main`
- blob SHA: `d548b565ebdac837dd77839e0d667ec9c5b79433`
- дата аудита: 2026-07-20

## Что это на самом деле

Скрипт не содержит собственной сборки Xray/3x-ui. Он:

1. Скачивает последнюю либо указанную версию официального `MHSanaei/3x-ui`.
2. Устанавливает официальный архив панели и официальный `x-ui.sh`.
3. Напрямую заполняет `/etc/x-ui/x-ui.db` через `sqlite3`.
4. Создаёт готовые inbounds: VLESS Reality, VLESS WS, VLESS XHTTP и Trojan.
5. Настраивает Nginx как SNI-мультиплексор на 443, TLS-сайт, панель и подписки.
6. Добавляет собственные компоненты проекта: Clash-шаблон, fake-site, LibreSpeed и Python MTR backend.
7. Включает UFW, системные тюнинги и cron-задачи.

Итого: это действительно **автоматическая обвязка вокруг официального 3x-ui**, а не подменённый бинарник панели.

## Критические и существенные замечания

### 1. Публичный прокси к произвольному localhost-порту

В Nginx создаётся location вида:

```nginx
location ~ ^/(?<fwdport>\d+)/(?<fwdpath>.*)$ {
    grpc_pass grpc://127.0.0.1:$fwdport$is_args$args;
    proxy_pass http://127.0.0.1:$fwdport$is_args$args;
}
```

Он не требует авторизации и позволяет внешнему клиенту обращаться через публичный домен к произвольным TCP/HTTP/gRPC-сервисам на `127.0.0.1`, зная либо перебирая порт. Это SSRF / localhost port-proxy. Случайный порт панели снижает удобство атаки, но не устраняет её: диапазон можно сканировать по различиям ответов.

**Рекомендация:** удалить generic location полностью. Для каждого транспорта создать отдельный точный `location` с жёстко заданным backend-портом и path.

### 2. Уничтожение существующей конфигурации без бэкапа

`clean_previous_install()` удаляет:

- `/etc/x-ui`
- `/usr/local/x-ui`
- все `/etc/nginx/sites-enabled/*`
- все `/etc/nginx/sites-available/*`
- все `/etc/nginx/stream-enabled/*`

При повторном запуске существующая база клиентов и вся конфигурация Nginx исчезнут. Скрипт рассчитан только на чистую, одноцелевую VPS.

**Рекомендация:** перед удалением делать timestamped backup и требовать явный флаг `--force-clean`.

### 3. Firewall может отрезать нестандартный SSH

Скрипт выключает UFW, затем разрешает только `22/tcp`, `80/tcp`, `443/tcp`, `443/udp` и включает UFW. Если SSH работает не на 22 порту, соединение будет потеряно.

**Рекомендация:** определять фактический порт `sshd` или не менять firewall без явного параметра.

### 4. Непривязанные к версии загрузки с правами root

Официальный 3x-ui скачивается как `latest`, без проверки SHA256/signature. Дополнительные Python/HTML/JS/YAML-файлы скачиваются из изменяемой ветки `mozaroc/3x-ui-pro/main`. Python backend затем запускается systemd-службой.

Это не выглядит как намеренный бэкдор, но создаёт обычный supply-chain риск: сегодняшний просмотр кода ничего не гарантирует для завтрашнего `curl | bash`.

**Рекомендация:** закрепить commit SHA и версии, хранить хэши всех загружаемых файлов, проверять их перед установкой.

### 5. Прямое редактирование SQLite-схемы 3x-ui

Скрипт напрямую вставляет настройки, inbounds и hosts в `/etc/x-ui/x-ui.db`. Это хрупко при изменениях схемы панели. Частичная проверка `group_id` есть, но полноценной совместимости по версиям нет.

**Рекомендация:** закрепить проверенную версию 3x-ui или использовать API/CLI панели.

## Прочие замечания

- Проверка `QEMU virtual CPU` и полный отказ установки выглядят необоснованно. Xray обычно не требует `host-passthrough` как обязательное условие.
- Создаётся тестовый файл на 1 GiB для LibreSpeed.
- Добавляется ежедневный рестарт `x-ui`.
- Сертификаты обновляются отдельной ежемесячной cron-задачей с остановкой Nginx, вместо штатного systemd timer Certbot.
- Используется внешний домен `cdn-one.org` в режиме auto-domain, то есть появляется зависимость от чужой DNS-зоны.
- Скрипт удаляет и переустанавливает Nginx при uninstall, включая `/etc/nginx` целиком.
- Входные параметры доменов валидируются слабо, хотя они затем попадают в Nginx-конфигурацию и SQL here-doc.

## Что выглядит нормально

- Бинарники панели и Xray берутся из официальных релизов `MHSanaei/3x-ui`.
- Случайные логин, пароль, paths, порты, Reality keys и short IDs создаются локально.
- MTR backend запускает `mtr` без `shell=True`, валидирует IP и не принимает произвольную цель от пользователя.
- Диагностический backend слушает только `127.0.0.1` и запускается от отдельного системного пользователя.
- Явных механизмов отправки паролей, ключей или базы третьей стороне в просмотренном коде не найдено.

## Вердикт

Скрипт не похож на вредоносный и действительно автоматизирует официальный 3x-ui. Но **в текущем виде ставить его на рабочую VPS без правок не следует** из-за публичного generic localhost proxy, разрушительной переустановки и неприкреплённых зависимостей.

Для одноразовой чистой VPS безопаснее сделать наш форк, убрать generic proxy и диагностику, закрепить версии и добавить резервное копирование.
